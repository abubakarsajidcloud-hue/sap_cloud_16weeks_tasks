# Toyota ERP Process Diagram Project

## Project Overview

This project demonstrates how an Enterprise Resource Planning (ERP) system integrates multiple business departments into a single platform.

The case study is based on a Toyota-style automotive manufacturing environment where customer orders, production planning, inventory management, finance operations, and logistics activities are connected through SAP ERP.

The purpose of this project is to understand:

* ERP fundamentals
* Enterprise business processes
* SAP integration concepts
* SAP Basis relevance
* SAP Cloud deployment concepts
* End-to-end transaction flow

---

# Business Problem

Before ERP implementation, departments often use separate applications:

| Department  | Tool                    |
| ----------- | ----------------------- |
| Procurement | Excel                   |
| Warehouse   | Legacy Inventory System |
| Finance     | Accounting Software     |
| HR          | HR System               |
| Sales       | CRM                     |

This creates:

* Duplicate data
* Delayed reporting
* Reconciliation effort
* Poor visibility
* Increased operational cost

---

# ERP Solution

SAP ERP provides:

* Single Database
* Real-Time Data
* Integrated Business Processes
* Centralized Reporting
* Better Decision Making

## PP (Production Planning)

* Production scheduling

## MM (Materials Management)

Responsible for:

* Procurement
* Inventory management
* Material tracking

## FI (Financial Accounting)

Responsible for:

* Revenue posting
* Cost tracking
* Financial reporting
